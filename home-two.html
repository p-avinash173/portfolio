Problem statement:


To create an online quiz portal with multiple REST APIs where users can browse different quizzes, attempt them, and find their scores and standings.

Scenario: This project, which aims to create an online quiz application, is based on different REST APIs through which users can log in and take quizzes. After finishing the quiz, they can find their standing and scores based on their accurate responses.
There are two types of users on this quiz portal:

  1. user
  2. Participants

Project Features:

  1. Includes a RESTful Web API to perform CRUD operations on Domain objects as per requirement using Spring Boot and MySQL/Oracle database.
  2. The admin user has a separate API to access the admin portal, which requires authentication with the admin username and password.
  3. The admin user can update the profile details and change the password after using the access token generated by login.
  4. The admin user can add questions using the addQuestions API.
  5. The admin user can create a quiz by entering quizid and selecting questions with questionid.
  6. The admin user can obtain statistics on total quizzes, questions, and users by using their APIs.
  7. The admin user can find the users who participated in the quiz along with scores and standings.
  8. Users can explore various quizzes created by the admin.
  9. Users can create their profile using new user registration.
  10. Users can take the quiz and try to answer the questions.
  11. Users can also check if the given answers are correct, as well as their results and positions.

  • Postman
  • Tomcat server
  • MySQL

Admin User Scenario:

  1. The admin wants to create an online quiz for the website users.
  2. The admin creates a set of questions along with their answers.
  3. Admin opens login API and logs in with the admin username and password.
  4. Once admin is authenticated, an access token is generated that can be used to add and modify quizzes, questions, and users.
  5. The admin creates a quiz by providing a name and an id.
  6. For creating a new quiz, the admin user enters a quizid and selects questions from the database using the questionid.
  7. After adding the questions, the admin runs the API and makes the quiz available to its users.
  8. Once the quiz is released, website users can start taking it.

Participants Scenario:

  1. The user uses the register API to create an account, which provides an access token.
  2. After registering an account, the user logs in with the access token to take any quiz.
  3. The user browses various quizzes created by Admin using the getAllQuizzes API.
  4. The user attempts the quiz using quizid and gives the possible answers.
  5. The user views whether the provided answers are right or wrong. The correct answer is highlighted differently so that they are easily identified.
  6. After completing the quiz, the user checks the scores and compares their standings with other users.
